# Plan: Survivorship Bias video — hyperframes-only build

> Source script: `mychannel/output/script.json` (+ `script.md`, `script.manifest.json`)
> Output: `mychannel/video/survivorship-v2/`
> Approach: **hyperframes only** for v1 (no generative-AI video, no 3D). Publish → observe retention → iterate. Generative/3D inserts deferred to a later pass.

## Requirements (confirmed with user)

- **Full video built entirely in hyperframes** (HTML→MP4). ~27 min, 8 scenes (manifest seg-00…seg-07, 0–1643s).
- **Narration:** TTS auto-generated (`hyperframes-media tts`).
- **Design:** dark editorial motion-graphics (Sisyphus 55 / Einzelgänger register) — restrained palette, kinetic typography, SVG/data-driven motion.
- **Captions:** synced burned-in captions only. **No BGM, no SFX.**
- **Spec:** 1920×1080 / 30fps (confirm at init).
- **Hard constraint:** visuals must be RICH, not boring — every scene earns its runtime.

## Architecture

- One hyperframes project at `mychannel/video/survivorship-v2/`.
- `index.html` = master composition; one sub-composition per manifest segment (`scenes/seg-00-hook.html` … `seg-07-outro.html`) via `data-composition-src` + `<template>`.
- Shared motif primitives in `scenes/_motifs/` (bomber silhouette SVG, dot-accumulation, empty-mat, success-curve) reused/transformed across scenes — this is the recurring-motif richness mechanism.
- TTS per-segment audio drives true segment durations; manifest t_start/t_end get reconciled to **actual** after TTS (replaces estimates).
- Captions generated by `hyperframes-media transcribe` on the TTS audio → burned in, synced.
- Final assembly via ffmpeg concat of per-scene renders + audio mux.

## Richness mechanisms (anti-monotony — the core requirement)

1. **Beat density 5–15s** — no static hold longer than ~15s; every beat introduces motion or transform.
2. **Recurring motif transforms** — bomber / empty-mat / success-curve reappear and *change meaning* (ties to Okada principle 1 + 8: re-context as reversal & re-visit reward).
3. **Kinetic typography** — key phrases animate in as their own beats, not just captions.
4. **Marker highlight** — the one phrase that matters per beat gets visual emphasis.
5. **Data-in-motion** — every statistic animates (count-up, bar growth, dot scatter) rather than appearing static.
6. **Pacing gear-change** — deliberately fill the manifest's Pacing gap (7/8 segments missing, principle 6 = 0 coverage): each scene gets ≥1 tempo break (hard cut / silence-hold / acceleration).
7. **Scene-specific visual registers** — each phase has a distinct compositional treatment so the eye never settles into one look.
8. **Seeded particle backgrounds** — deterministic (mulberry32) ambient texture so dark frames never feel dead. No `Math.random`, no `repeat:-1`.

## Phase 1 — Foundations + HOOK vertical slice (GATE)

1. `npx hyperframes init` at `mychannel/video/survivorship-v2/`; confirm 1920×1080/30fps.
2. Author `design.md`: palette, type scale, motif inventory, motion grammar, pacing rules.
3. TTS voice test: generate seg-00 narration, pick voice, measure actual duration vs estimate.
4. Build motif primitives in `scenes/_motifs/`.
5. Build **seg-00 HOOK** fully (bomber silhouette + accumulating red dots + clean-area warning glow, per the one explicit `[Visual:]` directive), with captions.
6. Render seg-00 to MP4.
7. **GATE:** user reviews the hook render before any further scenes. Locks the visual language.

## Phase 2 — Per-scene production

Production order (hardest scene budgeted explicitly):
seg-01 → seg-02 → **seg-03 (longest, ~360s — Industries, layered)** → seg-04 → seg-05 → seg-06 → seg-07.

Each scene: author HTML → lint/validate → TTS → caption sync → render → **individual commit**. Reconcile manifest `duration: estimate → actual` per scene.

## Phase 3 — Assembly & QA

- ffmpeg concat all scene renders; mux audio.
- QA: continuity of motifs, caption sync drift, no jump cuts (scene transitions present), determinism re-render check.
- Fill manifest actuals + write video metadata (title/description/tags from script).
- Final commit. (Publish + push only on explicit user instruction.)

## Risks

| Risk | Severity | Mitigation |
|---|---|---|
| Scope spans multiple sessions | HIGH | Per-scene commits; hook gate first; resumable via session file |
| TTS duration drifts from estimates | HIGH | Measure at Phase 1; reconcile manifest actuals per scene; animations keyed to audio |
| Caption sync accuracy | MED | Use transcribe word timings; QA pass for drift |
| Render memory on 27-min composite | MED | Render per-scene, concat with ffmpeg (not one giant render) |
| Monotony across 27 min | MED | 8 richness mechanisms above; per-scene visual registers |
| Non-determinism | MED | mulberry32 seeded PRNG only; no Math.random / repeat:-1 |

## Complexity: HIGH (multi-session)

## Acceptance criteria

- All 8 scenes render cleanly and concat into one ~27-min 1920×1080/30fps MP4.
- TTS narration + synced burned-in captions; no BGM/SFX.
- Every scene meets richness bar (no >15s static hold; ≥1 gear-change each).
- Manifest reconciled to actual durations.
- Deterministic re-render produces identical output.
