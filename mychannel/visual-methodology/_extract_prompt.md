# Extract Prompt — 岡田斗司夫 Clipping から方法論抽出

## あなたの役割
あなたは映像演出論の analyst。岡田斗司夫が YouTube 動画で語った内容の transcript (Obsidian Clipping) を読み、
「面白い映像 / 物語 / 演出を作るための **再現可能な方法論**」だけを抽出する。

## 重要な filter 条件
- **採用**: 他作品にも応用可能な技法・原則・分析 frame
- **採用**: 岡田が独自に命名した分析 frame (「○○の法則」「○○の構造」など)
- **採用**: 視聴者の感情を動かす仕組みの言語化
- **不採用**: 岡田の脱線・雑談・宣伝・タイムスタンプ
- **不採用**: 単に「面白い」「凄い」だけで根拠のない感想
- **不採用**: 作品のあらすじ要約 (重要なネタバレ含む構造分析は別)

## 出力形式 (厳守)

各 file につき 1 つの markdown を生成し、`mychannel/visual-methodology/extracted/{filename}.md` に保存。
filename は元 file 名から `.md` を取り、簡潔に短縮可 (例: `火垂るの墓_UG226.md`)。

```markdown
---
source: {元 file 名 (フルパス推奨)}
title: {作品名 or 主題}
director_or_author: {監督/著者}
year: {制作年/出版年、不明なら空欄}
extracted_at: 2026-05-30
---

# {作品名}

## 1行サマリ
{岡田の解説の核心を1文で}

## 岡田の核心テーゼ (なぜ刺さるか)
{なぜこの作品が面白い/凄いのか、岡田の中心的主張を 2-4 段落で}

## 演出技法 (具体)
### キャラ造形 / 表情 / シンボル
- {技法}: {具体的な使われ方} → {応用 hint}
- ...

### 色 / 光 / 構図 / カット
- {技法}: {具体的な使われ方} → {応用 hint}
- ...

### 音 / 間 / リズム
- {技法}: {具体的な使われ方} → {応用 hint}
- ...

## 物語構造の技法
- **起承転結の崩し**: ...
- **伏線**: ...
- **メタファー**: ...
- **メタ構造 (作中作・劇中の視点)**: ...
- **ダブルミーニング (表層と深層)**: ...

## 視聴者心理操作
- **期待裏切り**: ...
- **共感誘導**: ...
- **不快感制御**: ...
- **解釈余白 / 知識補完欲求**: ...

## 岡田 frame (独自命名分析)
- **{frame 名}**: {定義} — {この作品での適用例} — {他作品への汎用化}
- ...

## 引用 (鋭い 1-3 文)
> {岡田の発言の中で特に鋭い 1-3 文。要約不可、ほぼ原文}

## 応用 hint (この記事から学べる転用)
- 心理学解説動画なら: ...
- 抽象テーマを扱うなら: ...
- 短尺なら: ...
```

## 不明な場合の扱い
- 該当 section に岡田が言及していない場合: `- (該当言及なし)` と明記し、空欄にしない
- file 自体が方法論より作品紹介寄りの場合: 全 section をできる限り埋め、最後に `## 注記: この source は方法論抽出には情報量少` と末尾追記

## 量の目安
- 各 markdown は **400-1500 行**。短すぎ (200行未満) は抽出不足、長すぎ (2000行超) は要約不足。
- 引用は短く、応用 hint は具体的に。
